## Lời Mở Đầu

Chào mừng bạn đến với bài lab thực hành về **Machine Learning và Generative AI** trên nền tảng Amazon Web Services (AWS). Bài lab này sẽ hướng dẫn bạn từng bước cách tạo Customer Segment bằng RFM/ K-Means và xây dựng Agent để đưa ra "next best offer" bằng cách kết hợp sức mạnh của **AWS SageMaker AI** (nền tảng học máy) và **AWS Bedrock AI Agent** (nền tảng dịch vụ AI).

Thông qua bài lab này, bạn sẽ hiểu rõ hơn về cách hoạt động của các mô hình AI generative và làm quen với các công cụ AWS hiện đại nhất.

**Giải thích các thành phần chính:**

- **AWS S3**: Lưu trữ dữ liệu trên AWS
- **AWS Lambda Function**: Dịch vụ tính toán không máy chủ (serverless) để chạy code mà không cần quản lý server, thích hợp cho xử lý dữ liệu real-time và tích hợp các dịch vụ AWS với nhau.

## Kiến Thức Nền Tảng Liên Quan

Để tận dụng tối đa bài lab này, bạn nên làm quen với các khái niệm sau:

### 1. **Trí Tuệ Nhân Tạo Generative (Generative AI)**
- Khái niệm: AI generative là loại AI có khả năng tạo ra nội dung mới (văn bản, hình ảnh, mã code, v.v.)
- Ứng dụng: Chatbot, tạo văn bản, phân tích tài liệu, dịch thuật tự động

### 2. **Large Language Models (LLMs)**
- Mô hình ngôn ngữ lớn được đào tạo trên lượng dữ liệu khổng lồ
- Ví dụ: GPT, Claude, Llama, Mistral
- Khả năng: Hiểu và tạo ra văn bản giống con người

### 3. **AWS SageMaker**
- Nền tảng học máy được quản lý hoàn toàn của AWS
- Các tính năng chính: Notebook instances, training jobs, model deployment, monitoring
- Lợi ích: Đơn giản hóa quy trình xây dựng mô hình ML

### 4. **AWS Bedrock**
- Dịch vụ cung cấp các mô hình nền tảng sẵn dùng
- Không cần huấn luyện mô hình, chỉ cần gọi API
- Hỗ trợ nhiều mô hình từ các nhà cung cấp khác nhau

### 5. **Python & Jupyter Notebooks**
- Python: Ngôn ngữ lập trình phổ biến trong data science
- Jupyter: Công cụ tương tác để viết và thực thi code
